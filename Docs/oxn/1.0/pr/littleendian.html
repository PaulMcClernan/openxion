Properties

littleEndian

Supported By

  • XION Scripting Language Standard (1.0)
  • OpenXION (1.0, 1.1, 1.2, 1.3, 1.4)
  • OpenXION Standard Module (1.0, 1.1, 1.2, 1.3, 1.4)

Applies To

Global environment

Examples

set the littleEndian to true -- use little-endian byte order

set the littleEndian to false -- use big-endian byte order

Description

The littleEndian property determines or changes the endianness of the tinyInt, shortInt, mediumInt, longInt, halfFloat, singleFloat, and doubleFloat binary numeric chunk types. A true value indicates that binary data should be treated as little-endian. A false value indicates that binary data should be treated as big-endian. The default is false, or big-endian. (This was chosen as the default because the 68000 and PowerPC architectures running Mac OS Classic and the Java virtual machine are big-endian.)
